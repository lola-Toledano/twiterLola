# TwitterLola 🚀

TwitterLola es una red social minimalista y moderna construida con **Vanilla JavaScript** y **Supabase**. No requiere Node, ni NPM, ni procesos de compilación complejos. ¡Pura web! 🐾

## 🛠️ Tecnologías
- **Frontend**: HTML5, CSS3 (Glassmorphism), JavaScript (ES6+).
- **Backend-as-a-Service**: Supabase.
- **Despliegue**: Cloudflare Pages + GitHub.

## 🚀 Instrucciones de Despliegue

### 1. Preparar el repositorio en GitHub
1. Entra en [GitHub](https://github.com/) y crea un nuevo repositorio llamado `TwitterLola`.
2. Sube tus archivos (`index.html`, `style.css`, `app.js`, `config.js`, `_redirects`).
   * *Truco*: Puedes arrastrar los archivos directamente a GitHub si no usas la terminal.

### 2. Desplegar en Cloudflare Pages
1. Entra en el panel de [Cloudflare](https://dash.cloudflare.com/) y ve a **Workers & Pages**.
2. Pulsa en **Create application** -> **Pages** -> **Connect to Git**.
3. Selecciona tu repositorio `TwitterLola`.
4. En los ajustes de build:
   - **Framework preset**: None.
   - **Build command**: (Dejar vacío).
   - **Build output directory**: (Dejar vacío o poner `./`).
5. Pulsa **Save and Deploy**. ¡En 1 minuto tu app estará en una URL pública!

## 💾 Configuración de la Base de Datos
Si estás montando esto de cero, ejecuta este SQL en el **SQL Editor** de Supabase:

```sql
CREATE TABLE posts (
  id UUID DEFAULT uuid_generate_v4() PRIMARY KEY,
  content TEXT NOT NULL,
  author_id TEXT NOT NULL,
  created_at TIMESTAMP WITH TIME ZONE DEFAULT now() NOT NULL
);

CREATE TABLE likes (
  id BIGINT GENERATED BY DEFAULT AS IDENTITY PRIMARY KEY,
  post_id UUID REFERENCES posts(id) ON DELETE CASCADE,
  device_id TEXT NOT NULL,
  UNIQUE(post_id, device_id)
);

ALTER TABLE posts ENABLE ROW LEVEL SECURITY;
ALTER TABLE likes ENABLE ROW LEVEL SECURITY;

CREATE POLICY "Lectura libre" ON posts FOR SELECT USING (true);
CREATE POLICY "Publicación libre" ON posts FOR INSERT WITH CHECK (true);
CREATE POLICY "Borrado autor" ON posts FOR DELETE USING (author_id = (SELECT author_id FROM posts WHERE id = posts.id));
CREATE POLICY "Lectura likes" ON likes FOR SELECT USING (true);
CREATE POLICY "Like libre" ON likes FOR INSERT WITH CHECK (true);
```

## 📝 Nota sobre `config.js`
Asegúrate de que tu `config.js` tenga la `SB_URL` y la `SB_KEY` de tu proyecto de Supabase para que todo funcione.
